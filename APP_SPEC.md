# アプリケーション設計書 — 論文和訳PDFツール GUI版

**版**: Draft v0.1(2026-07-18)
**前提**: 翻訳パイプラインの仕様は [SPEC.md](SPEC.md) に従う。本書はそれを「アプリ」として
化学系の非プログラマに配布・利用してもらうためのUI/アーキテクチャ設計を定める。
この段階では実装は行わない。

---

## 1. アプリ形態の選定

| 案 | 概要 | 評価 |
|---|---|---|
| **A. ローカルWebアプリ(採用)** | FastAPI + ブラウザUI。PyInstallerで単一exe化し、起動するとブラウザが開く | UIが作りやすくPDFプレビューも容易。配布はexe 1個+フォルダ。**推奨** |
| B. デスクトップGUI (Tkinter/Qt) | ネイティブウィンドウ | PDFプレビューや対訳レビュー画面の実装コストが高い |
| C. クラウドWebサービス | サーバーにデプロイし全員がブラウザで使う | 運用者が必要・論文PDFを外部サーバーに置くことになる(未公開データの扱いで不利)。将来案 |

**採用: 案A**。すべてローカルで動くため、未発表原稿や査読中PDFを外部に送るのは
翻訳APIへのテキスト送信のみ(この点はUI上に明示する。§5.6)。

## 2. 全体アーキテクチャ

```
┌─────────────────────────── ローカルPC ───────────────────────────┐
│  wayaku.exe (PyInstaller)                                        │
│  ├── FastAPI サーバー (127.0.0.1:自動割当ポート)                  │
│  │    ├── /api/*      … ジョブ管理・翻訳・設定 REST API           │
│  │    └── /           … フロントエンド静的ファイル配信            │
│  ├── コアエンジン(SPEC.md の①〜⑥パイプライン)                   │
│  │    └── バックグラウンドワーカー(ジョブキュー、1並列)          │
│  └── 起動時に既定ブラウザを自動オープン                           │
│                                                                  │
│  データディレクトリ: %APPDATA%/WayakuSakusei/                     │
│  ├── settings.json   … APIキー(DPAPIで暗号化)・既定エンジン等    │
│  ├── glossaries/     … 内蔵+ユーザー用語集                       │
│  ├── jobs/<job_id>/  … 入力PDF・中間データ・出力PDF・report.json  │
│  └── cache/          … 翻訳キャッシュ(SPEC.md §9)               │
└──────────────────────────────────────────────────────────────────┘
                     │ HTTPS(テキストのみ)
                     ▼
          翻訳API (Claude / DeepL / Google)
```

- フロントエンドは素のHTML+JS(または軽量にVue/Reactビルド済み静的ファイル)。ビルド産物をexeに同梱
- CLI(main.py)は残し、GUI/CLIどちらも同じコアエンジンを呼ぶ(ロジックの二重実装禁止)
- サーバーは `127.0.0.1` のみバインド。外部からアクセス不可

## 3. 画面設計

### 3.1 画面一覧と遷移

```
[ホーム(ジョブ一覧)] ─→ [新規翻訳ウィザード] ─→ [進捗画面] ─→ [レビュー画面] ─→ [完了/ダウンロード]
        │
        └→ [設定画面] / [用語集管理画面]
```

### 3.2 ホーム(ジョブ一覧)
- 過去ジョブのカード一覧: 論文ファイル名、日時、状態(完了/処理中/エラー/レビュー待ち)、使用エンジン
- 「新しいPDFを翻訳」ボタン(またはページ全体へのドラッグ&ドロップ)
- ジョブの再開・削除・出力PDFの再ダウンロード

### 3.3 新規翻訳ウィザード(3ステップ)
1. **PDF選択**: ドラッグ&ドロップ or ファイル選択。アップロード直後にサーバー側で検査し、
   スキャンPDF(テキスト層なし)の場合はここでエラー表示
2. **翻訳設定**:
   - エンジン選択(claude / deepl / google。APIキー未設定のエンジンはグレーアウトし設定画面へ誘導)
   - 分野選択(有機/無機/分析/物理/高分子/材料/一般)→ SPEC.md §4.2 の分野別用語集に対応
   - 対象ページ範囲、参考文献を訳すか、追加用語集の選択
   - **要約オプション**(SPEC.md §10): 「要約を付ける」チェック+長さ(短/標準/詳細)。
     「要約のみ(全文翻訳なし)」モードも選択可 — まず要約で読む価値を判断し、必要なら同じジョブから全文翻訳へ進める
3. **確認**: 抽出プレビュー(dry-run結果。ブロック分類を色分け表示: 本文=青、保護=黄、除外=灰)
   +推定コスト(文字数からの概算)を表示 → 「翻訳開始」

### 3.4 進捗画面
- 進捗バー(ブロック単位: 翻訳済み n/N)+段階表示(抽出→分類→翻訳→検証→書き込み)
- キャンセルボタン(翻訳済みブロックはキャッシュに残り、再実行時に再利用)
- エラー時: 原因(APIキー無効・レート制限・ネットワーク)を平易な日本語で表示し、リトライボタン

### 3.5 レビュー画面(化学系ユーザー向けの要)
書き込み前に訳文を人間が確認・修正できる画面。SPEC.md §7 の検証結果と連動する。

- **左**: 元PDFのページ画像(該当ブロックをハイライト)
- **右**: ブロックごとの「原文/訳文」ペアのリスト。訳文はその場で編集可能
- フィルタ: 「要注意のみ表示」(数値欠落・長さ異常・分類不確実のフラグ付きブロック)
- 用語ハイライト: 用語集にヒットした語を訳文中で下線表示し、ホバーで対応する英語を表示
- 「この訳語を用語集に登録」: 訳文中の語を選択→英語対応を入力→ユーザー用語集に追加(次回以降に反映)
- 「承認して書き込み」ボタンで⑥のPDF書き込みを実行。レビューをスキップする設定も可(既定はレビューあり)

### 3.6 完了画面
- 出力PDFのプレビューとダウンロード(既定ではジョブフォルダに保存済み。「フォルダを開く」ボタン)
- **要約の表示**(要約オプション有効時): 構造化要約(目的/手法/結果/結論/キーワード)を画面に表示し、
  コピー・Markdownダウンロード可。「要約のみ」ジョブの場合はここから「全文翻訳に進む」ボタンで同一ジョブを翻訳へ移行
- report.json の要約表示: 翻訳ブロック数、スキップ一覧、使用した訳語対応表、API使用量概算
- 対訳TSVのダウンロード

### 3.7 設定画面
- APIキー入力(エンジンごと)。**入力されたキーはOSの資格情報保護(Windows DPAPI)で暗号化して
  settings.json に保存**し、画面上はマスク表示。「接続テスト」ボタンで疎通確認
- 既定エンジン・既定分野・レビュー画面の有無・フォント選択
- 日本語フォント未配置の場合はここで警告し、Noto Sans JP のダウンロード案内(自動ダウンロードボタンも検討)

### 3.8 用語集管理画面
- 内蔵用語集(読み取り専用)とユーザー用語集(編集可)をタブ表示
- 表形式で追加・編集・削除、TSVのインポート/エクスポート
- 検索ボックス(英語・日本語どちらでも)

## 4. REST API 設計(概要)

| メソッド | パス | 機能 |
|---|---|---|
| POST | /api/jobs | PDFアップロード+設定でジョブ作成(multipart) |
| GET | /api/jobs | ジョブ一覧 |
| GET | /api/jobs/{id} | ジョブ詳細+進捗(ポーリング。将来SSE化) |
| POST | /api/jobs/{id}/start | dry-run確認後に翻訳開始 |
| POST | /api/jobs/{id}/cancel | キャンセル |
| GET | /api/jobs/{id}/blocks | レビュー用ブロック一覧(原文・訳文・フラグ) |
| PATCH | /api/jobs/{id}/blocks/{bid} | 訳文の手動修正 |
| POST | /api/jobs/{id}/finalize | 承認→PDF書き込み実行 |
| GET | /api/jobs/{id}/result | 出力PDF取得 |
| GET | /api/jobs/{id}/summary | 要約取得(Markdown/JSON) |
| POST | /api/jobs/{id}/translate-full | 「要約のみ」ジョブを全文翻訳へ移行 |
| GET/PUT | /api/settings | 設定の取得・保存 |
| POST | /api/settings/test-engine | APIキー疎通テスト |
| GET/POST/DELETE | /api/glossary/... | 用語集CRUD・インポート/エクスポート |
| GET | /api/jobs/{id}/pages/{n}.png | レビュー画面用ページ画像(PyMuPDFでレンダリング) |

## 5. 非機能・配布要件

### 5.1 配布物
- `WayakuSakusei-Setup.exe`(インストーラ、Inno Setup等)または zip 展開型の `wayaku.exe`
- 同梱: フロントエンド静的ファイル、内蔵用語集TSV、README(化学系ユーザー向けの平易な導入手順)
- 日本語フォントはライセンス確認のうえ同梱可否を判断(Noto Sans JP は OFL のため同梱可)

### 5.2 対応環境
- Windows 10/11 を主対象(利用者環境)。コアはクロスプラットフォームに保ち、macOS対応は将来
- オフライン動作: 翻訳API以外はネットワーク不要

### 5.3 更新
- 起動時にGitHub Releases の最新バージョンを確認し、新版があれば通知のみ(自動更新はしない)

### 5.4 多重起動・排他
- 二重起動時は既存プロセスのブラウザタブを開く(ポートファイルで検出)
- ジョブは1並列で直列処理(APIレート制限とメモリの単純化のため)

### 5.5 エラー表示ポリシー
- 利用者は非プログラマ。スタックトレースは画面に出さず、`jobs/<id>/error.log` に保存。
  画面には「原因+ユーザーが取れる対処」を日本語で表示

### 5.6 プライバシー表示
- 初回起動時とジョブ確認画面に「本文テキストは選択した翻訳APIの提供者に送信されます」と明示し、
  各社のデータ保持ポリシーへのリンクを置く(未公開原稿を扱う化学研究者への配慮)

## 6. 既存草案からの変更点

- ディレクトリ再編:
  ```
  wayaku_sakusei/
  ├── core/                  # SPEC.md のパイプライン(現 pdf_translator.py + translators/ を移動)
  ├── server/                # FastAPI アプリ・ジョブ管理・設定管理
  ├── frontend/              # UI(ビルド産物を dist/ に)
  ├── glossaries/
  ├── main.py                # CLI(core を直接呼ぶ)

  ```
